## Implementation Plan

### Phase 1: Fix Preview Bug (Critical)
**File**: `frontend/src/components/Preview.vue`

The preview currently injects `<style>` tag but does NOT wrap content in template container. This means template CSS selectors (`.overwatch-template h1`, etc.) don't match.

**Fix**: Wrap the content div in a template container when a template is active.

```vue
<template>
  <div ref="previewRef" class="h-full overflow-auto p-8 bg-white dark:bg-[#0d1117]">
    <div v-if="activeTemplate" v-html="templateStyle"></div>
    <div 
      ref="contentRef"
      class="markdown-body mx-auto max-w-4xl"
      :class="activeTemplate ? `${activeTemplate.id}-template` : ''"
      v-html="html"
    />
  </div>
</template>
```

## Key Design Decisions

1. **Light Theme**: User requested light theme (not dark), so we use white background with dark text
2. **Comprehensive Coverage**: Implement ALL 65 CSS classes for complete style coverage
3. **Responsive Design**: Include media queries for all breakpoints (720px, 960px, 1200px, 1400px)
4. **Font Fallback**: Use "Config" with fallback to "Segoe UI", Arial, sans-serif
5. **Orange Accents**: Use Overwatch's signature orange (#f06414, #ffab2e) for links and borders
